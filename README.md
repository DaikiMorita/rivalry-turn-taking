# 司会者なしで「発言の椅子」を回す ── 分散ターンテイキングの最小デモ

複数のエージェントが同じ場に居続けるとき、「**誰が、いつ話すのか**」を、中央の進行役
（司会者）を置かずに、各エージェントの自律性を保ったまま決められるか。脳の
**両眼視野闘争（binocular rivalry）** の力学を借りて、それを最小コードで示します。

- **標準ライブラリ（`math` / `random`）だけ**で動きます。特定のアプリには依存しません。
- 各エージェントの刺激 `u_i` が**どこから来るか（言語モデル / ルール / 人間）は問いません**。

> これは本番プロダクトの分散ターンテイキング・エンジンの**中核だけを取り出した教育用 mock** です。
> 本番の drive 計算（公開された事実から刺激 u を作る部分）・空間配線・調整済みの本番値・各種
> ガードは含みません。目的は原理を再現可能な最小形で示すことです。

---

## このデモが見せる2つのこと

### 柱1. これは「rivalry という名のオーケストレーター」ではない ── peer-mesh（分散）である

よくある疑問: 「rivalry と言っても、結局は全員を見て次の話者を選ぶ"中央の司会者"では？」
答えは No で、それを**コードの構造**が満たす2つの不変条件で示します（`rivalry.py`）。

1. **共有バスに乗るのは公開情報だけ。** 各ノードが場に出す「公開出力（占有度への寄与）」の
   合計＝場の占有度 `y` と、「誰がいつ閾値を越えたか」という公開イベント。**他ノードの私的状態
   `(x,a,b)` も発話内容も、誰も読みません。** 実際 `RivalryNode.step(field_y, drive_u, rng)` の
   引数は「自分の状態＋公開スカラー＋自分の drive」だけで、他ノードのオブジェクトを受け取りません
   ＝構造的に他者の内部を読めない。
   （`y` は「中央が私的状態を集めて回る」のではなく、各自が公開出力を場に出し各自が合計を感じる
   **平均場結合**。蛍の同期・クオラムセンシングと同型の、指揮者なしの集団力学です。）

2. **勝者は"審議する権威"でなく"全員共有の固定ルール"で決まる。** `decide_winner()` は公開イベント
   （誰がいつ越えたか）だけの純関数で、文脈も中身も読みません。各ノードがこの関数をローカルに
   実行しても同じ勝者を得ます＝中央の権威は不要。1か所で `min()` を呼ぶのは分散合意ルールの中央
   "実装"にすぎず、photo-finish のカメラのように「勝者を決める」のでなく「創発した結果を記録する」だけ。

つまり who-talks は《決定》でなく《創発（first-passage）》。脳の両眼視野闘争に「どちらの像を
見せるか決める小人」がいないのと同じ構図を、コードで保っています。

### 柱2. 健全な会話は力学の『窓』に宿る

このモデルは**パラメータ次第で何にでもなります** ── 一体が永遠に話し続ける独占にも、全員が黙る
沈黙にも、健全な交代にも。だから「交代するように調整した」では弱い。見せたいのは、**健全な会話
（独占せず・名指しに応え・無駄に喋らない）が"偶然"ではなく、相互抑制・疲労・揺らぎのバランスが
取れた『窓』にしか宿らない**こと。全レンジを正直に見せ、その窓を特定します（`experiments.py` の [A][B]、および `sigma` を振る実験）。

## ファイル

| ファイル | 中身 |
|---|---|
| `rivalry.py` | 中核ロジック（ライブラリ）。自律ノード `RivalryNode`／場の公開スカラー結合／`decide_winner`（公開イベントだけの固定ルール）。peer-mesh の不変条件はここ。**`experiments.py` から import されるだけで、単体では実行しない。** |
| `experiments.py` | **実行する入口（これを動かす）。** 状態（独占/沈黙/健全）／窓／baselines 比較／N=2・N=3 時系列を再生成。 |
| `tests/` | 最小テスト（標準ライブラリ `unittest`）。peer-mesh の不変条件と first-passage の勝者ルール、独占/健全の状態を固定する。 |

## 動かし方

```bash
python3 experiments.py
```

動かすのは `experiments.py` だけです。`rivalry.py` は中核ロジックで、`experiments.py` から読み込まれます（`rivalry.py` を直接実行しても何も起きません）。

依存パッケージなし（標準ライブラリ `math` / `random` のみ）。**動作確認: Python 3.12.8（macOS）**。3.10+ 専用の構文は使っていないので Python 3.8 以上で動く想定です。20 seed × 各 2000 ターンの平均を 10 秒ほどで出します。

## テスト

```bash
python3 -m unittest discover -s tests -v
```

これも標準ライブラリ（`unittest`）だけで動きます（追加インストール不要）。docstring や名前ではなく**実行できる検査**で、README が主張する2つのこと ──「中央の司会者がいない（peer-mesh）」と「first-passage の勝者ルール」、そして「同じ力学が独占にも健全な交代にもなる」── を最小限で固定しています。

## 各セクションの読み方

- **[A] 状態**: 同じ力学が独占・沈黙・健全のどれにもなる。独占は「独占を防ぐ機構（不応・疲労・揺らぎ）を切った」極。健全はフル機構で、N=2（きれいな左右交互＝両眼視野闘争そのもの）と N=3（組が入れ替わって全員に椅子が回る＝2 体で当然のことが 3 体でも成立）で見せる。
- **[B] 名指し sweep + 比較**: 一者の刺激を上げると share は滑らかに増すが 0.5 で頭打ち（独占できない）。round-robin は名指しを無視（0.33 固定）、weighted-random は連続を止められない（4%）。「名指しに応える(0.49) ∧ 独占させない(0%)」を同時に満たすのは rivalry だけ。
- **揺らぎの窓**: `experiments.py` の `DEFAULTS` で `sigma` を振ると、0=勝者総取り → 0.05〜0.1=健全 → 0.4=劣化。独占を防ぐのは不応・疲労、対称を破るのは揺らぎ。両方がちょうどよい範囲＝『窓』。

## 本番との違い（このデモの簡約点）

原理を見せるための簡約形です。本番エンジンとの主な違いは次のとおりです。(a) 相互抑制の項に、はっきり
立ち上がった者だけを強く数える急峻な占有度関数と、在席数によらない一定の基底抑制を足している。(b) 活性
`x`・遅い疲れ `a` のほかに、もっとゆっくり動く「気分」変数と、時間帯で活性のベースが上下する日内リズムの
項を持つ。(c) 速い不応 `b` は本番では毎ターン使い捨てだが、このデモでは話を簡単にするためノードが保持する。
これらの補助項を省いていますが、ODE 定数は本番と同じです。
