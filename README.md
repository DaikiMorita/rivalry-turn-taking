# 司会者なしで「発言の椅子」を回す ── 両眼視野闘争の最小デモ

複数のエージェントが同じ場に居続けるとき、「**誰が、いつ話すのか**」をどう決めるか。
中央の進行役（司会者）を置かずに、各エージェントの自律性を保ったまま、それでも会話の
順番が破綻なく回る ── そんな仕組みを、脳の **両眼視野闘争（binocular rivalry）** の
力学から借りて作った、最小の再現コードです。

- **標準ライブラリ（`math` / `random`）だけ**で動きます。特定のアプリ・フレームワークに
  一切依存しません。
- 各エージェントの外部刺激 `u_i` が**どこから来るか（言語モデル / ルール / 人間）は問いません**。
  この力学は「活性 `x_i` がしきい値を最初に越えたのは誰か」だけを見て発言者を決めます。

> これは記事の実証パート（モデルの実証）に対応する玩具シミュレータです。**原理が伝わる
> ことを目的にした簡約形**で、実プロダクトの本番エンジンそのものではありません（ODE 定数は
> 同じですが、相互抑制の細かな作り込みは省いています。下記「本番との違い」参照）。

## 何を示すか

会話で同時に話せるのは一人だけ ── この「発言の椅子」を、座りたいエージェントたちが
どう代わる代わる使うか。欲しい性質は3つ:

1. 同時に座れるのは一人だけ（**相互抑制**）
2. 一人が座り続けない（**疲労・不応**）
3. これを**中央の判定者なし**で実現する

両眼視野闘争（左右の目に違う絵を見せると、混ざらず・片方ずつ交互に見える現象）には、
この3つがそのまま入っています ── 互いに抑え合う / 勝っている側が疲れる / 少しの揺らぎ。
これを「発言の椅子」に重ねると、司会者なしで順番が回り始めます。

勝者は「ある瞬間の活性がいちばん高い者」ではなく、**ひと区切りのあいだに最初にしきい値を
越えた者**で決めます（first-passage）。両眼視野闘争型の式は本質的に振動するので、瞬間の
スナップショットだと位相でたまたま浮いた傍観者を拾ってしまうためです。

## 出力例（20 seeds × 2000 turns）

```

(1) 対等な ambient では黙る（沈黙率）
  2 体: 沈黙 87% / 話したときの share 0.51 / 0.49
  3 体: 沈黙 91% / 話したときの share 0.34 / 0.33 / 0.33
  4 体: 沈黙 96% / 話したときの share 0.26 / 0.23 / 0.25 / 0.26

(2) みなが乗っていれば均等に回り、独占しない（刺激を全員 1.0）
  3 体: 沈黙 2% / share 0.33 / 0.35 / 0.32 / 連続4回以上 0%

(3) 名指しは滑らかに効くが独占はできない（3 体・一者の刺激だけ上げる）
  刺激 | その者の share
  1.0  | 0.33
  1.2  | 0.44
  1.4  | 0.49
  1.6  | 0.50
  1.8  | 0.50
  2.0  | 0.50

(4) 揺らぎの役割（2 体・刺激同じ・sigma を変える）
  sigma=0.0 : share 1.00 / 0.00 / 連続4回以上 0%
  sigma=0.05: share 0.50 / 0.50 / 連続4回以上 0%
  sigma=0.1 : share 0.50 / 0.50 / 連続4回以上 0%
  sigma=0.4 : share 0.50 / 0.50 / 連続4回以上 7%

(5) 3 方式の比較（3 体・一者だけ刺激 1.4）
  方式         | 名指しの share | 連続4回以上
  rivalry      | 0.49           | 0%
  round-robin  | 0.33           | 0%
  weighted-rnd | 0.41           | 4%
```

（揺らぎは乱数なので 1〜2 ポイントは振れます。傾向は再現します。）

## 各実験の読み方

1. **対等な ambient では黙る**: 誰にも向けられていない弱い刺激だけだと、ほとんどのターンで
   誰も座りません。中央の進行役なら毎ターン必ず一人を指名してしまうところを、この仕組みは
   「今は誰も話さなくていい」を表現できます（しかも稀に話すときは均等）。
2. **乗っていれば均等に回り、独占しない**: 全員の刺激を上げると沈黙はほぼ消え、share は均等、
   同じ者の連続はほぼゼロ。たった今喋った者は不応で一拍抑えられ、椅子が別の者に渡るためです。
3. **名指しは滑らかに効くが独占できない**: 一者の刺激だけ上げると主導権は滑らかに増しますが、
   **0.5 あたりで頭打ち**。どれだけ強く名指されても、一度喋れば一拍抑えられるので「一回おき」が
   上限になります（＝独占を構造的に防ぐ）。
4. **揺らぎの役割**: `sigma=0` だと対称が破れず勝者総取り（1.0 / 0.0）。少し入れると 0.5 / 0.5 に
   均れ、入れすぎると今度は同じ者が続く方に寄ります。ちょうどよい範囲があります。
5. **3 方式の比較**: **round-robin** は「いま誰が話したいか」を完全に無視（名指しされても 0.33）。
   **weighted-random** は名指しを重みに反映できるが、連続を止められない。**rivalry だけ**が
   「名指しに応える（0.50）」と「独占させない（連続 0%）」を同時に満たします。

## 本番との違い（このデモの簡約点）

原理を見せるための簡約形です。実プロダクトの本番エンジンは、相互抑制の項に
(a) はっきり立ち上がった者だけを強く数える急峻な占有度関数、(b) 在席数によらない一定の基底
抑制、を足しています。また速い不応 `b` は本番では毎ターン使い捨てですが、このデモでは
話を簡単にするためターンを跨いで持ち越しています。ODE 定数（`alpha`/`w_s`/`w_I`/… ）は本番と同じです。
